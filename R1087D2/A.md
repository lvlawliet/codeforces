### 1. 题目翻译

**题目：A. Flip Flops (拖鞋)**

**题目描述：**
OtterZ 为了提升自己的战斗力，安排了一场与 $n$ 只怪物的战斗。每只怪物的战斗力为 $a_i$，OtterZ 的初始战斗力为 $c$。他手里有 $k$ 只拖鞋，并且可以执行以下两种操作：
1. **击杀**：如果一只存活怪物的战斗力 $a_i \le c$，OtterZ 可以击杀它。击杀后，OtterZ 的战斗力 $c$ 会增加，变为 $c + a_i$。
2. **激怒（扔拖鞋）**：向一只存活的怪物扔一只拖鞋。拖鞋会损坏，怪物会变得更加愤怒，导致其战斗力 $a_i$ 增加 1，变为 $a_i + 1$。

请你帮助 OtterZ 计算出战斗结束后，他能获得的最大战斗力 $c$ 是多少。

**输入格式：**
* 第一行包含测试用例的数量 $t$ ($1 \le t \le 500$)。
* 每个测试用例的第一行包含三个整数 $n, c, k$ ($1 \le n \le 100$， $0 \le c, k \le 10^9$)，分别代表怪物数量、初始战斗力、拖鞋数量。
* 第二行包含 $n$ 个整数 $a_1, a_2, \dots, a_n$ ($0 \le a_i \le 10^9$)，代表每只怪物的初始战斗力。

**输出格式：**
* 对于每个测试用例，输出一个整数，表示 OtterZ 可能达到的最大战斗力。

---

### 2. 解题思路

**算法剖析：**

这道题的核心机制在于**“扔拖鞋”的本质**。对怪物扔拖鞋会让它的战斗力 $+1$；如果我们之后成功击杀了这只怪物，那么这 $+1$ 的战斗力最终会转移到 OtterZ 身上。因此，**扔拖鞋可以看作是一种等价的“战力投资”**，前提是投资后我们依然能打得过它。

1.  **投掷上限分析**：为了能够击杀怪物 $i$，投资后它的战斗力不能超过我们当前的战斗力。所以，我们最多只能对它扔 `当前战斗力 - a[i]` 只拖鞋。
2.  **贪心策略（排序）**：我们应该先杀弱的怪物还是强的怪物？显然，先杀弱的怪物可以无风险地提升我们自己的战斗力。我们的战斗力越高，不仅能击杀更强的怪物，能给怪物“投资”的拖鞋数量上限（`当前战斗力 - a[i]`）也就越大。因此，**首先对怪物的初始战斗力进行升序排序**是最佳策略。
3.  **枚举击杀数量**：虽然我们想要尽可能多地投资并击杀怪物，但并不是击杀的怪物越多，最终战力就一定越高。有时候把拖鞋全投资在前面几个怪物身上，可能比留着拖鞋去勉强杀更多怪物收益更大。
    因此，代码采用了一种非常稳妥的枚举法：**枚举我们计划击杀的怪物总数 $m$（从 $1$ 到 $n$）**。
    * 对于每一个 $m$，我们只尝试击杀排序后的前 $m$ 只怪物。
    * 每次击杀时，只要手里还有拖鞋，就拉满投资，把能扔的拖鞋全扔给它（即取 `剩余拖鞋数` 和 `当前战力 - a[i]` 的较小值）。
    * 模拟完击杀这 $m$ 只怪物后，记录下此时的最终战斗力。如果中途发现某只怪物打不过，说明击杀 $m$ 只怪物的计划不可行。
    * 最后在所有可行的计划中，取最终战斗力的最大值。

**复杂度分析：**
* **时间复杂度**：对怪物数组排序需要 $O(n \log n)$。外层循环枚举击杀数量 $m$ 需要执行 $n$ 次，内层循环模拟击杀前 $m$ 只怪物最多也执行 $n$ 次。所以嵌套循环的复杂度是 $O(n^2)$。综合下来时间复杂度为 $O(n^2)$。由于 $n \le 100$，每个测试用例的操作次数最多只有 $10000$ 次左右，配合 $t \le 500$，总计算量在 $5 \times 10^6$ 级别，远远小于 C++ 一秒内 $10^8$ 的计算极限，运行非常快。
* **空间复杂度**： $O(n)$，仅使用了一个大小为 $n$ 的 `vector` 来存储怪物的战斗力。

### 3. 代码详解

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve()
{
    int n;
    long long c, k;
    cin >> n >> c >> k;
    
    // a 数组存储所有怪物的战斗力
    vector<long long> a(n);
    for (int i = 0; i < n; i++)
    {
        cin >> a[i];
    }
    
    // 贪心第一步：将怪物战斗力从小到大排序
    // 这样可以确保我们优先击杀弱小的怪物，滚雪球式积累自身战力
    sort(a.begin(), a.end());
    
    // 记录可能达到的最大战斗力，保底为初始战力 c
    long long ans = c;
    
    // 枚举我们要击杀的怪物数量 m，从 1 只到 n 只
    for (int m = 1; m <= n; m++)
    {
        long long cur = c;   // 当前情况下的初始战斗力
        long long rem = k;   // 当前情况下的可用拖鞋数
        bool ok = true;      // 标记是否能成功击杀前 m 只怪物
        
        // 模拟击杀前 m 只怪物
        for (int i = 0; i < m; i++)
        {
            // 如果遇到打不过的怪物，说明击杀前 m 只的计划失败
            if (a[i] > cur)
            {
                ok = false;
                break;
            }
            
            // 计算为了保证还能打过它，最多能给它扔多少只拖鞋
            long long can_add = cur - a[i];
            
            // 实际能扔的拖鞋数，取决于手里剩下的拖鞋和能扔的上限
            long long use = min(can_add, rem);
            
            // 击杀该怪物，战力增加：怪物原始战力 + 扔拖鞋提升的战力
            cur += a[i] + use;
            
            // 扣除消耗的拖鞋
            rem -= use;
        }
        
        // 如果成功击杀了这 m 只怪物，尝试更新历史最高战斗力
        if (ok)
        {
            ans = max(ans, cur);
        }
    }
    
    // 输出当前测试用例的最大战斗力
    cout << ans << '\n';
}

int main()
{
    // 优化输入输出流，提高读取速度
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int t;
    cin >> t;
    while (t--)
    {
        solve();
    }
    return 0;
}
```
