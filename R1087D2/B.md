### 1. 题目翻译

**题目：B. Array (数组)**

**题目描述：**
给你一个长度为 $n$ 的整数数组 $a$。
对于每一个下标 $i$，你需要找到满足以下条件的最大下标 $j$ 的数量：
* $j > i$
* $|a_i - k| > |a_j - k|$
这里的 $k$ 可以是**任意整数**。你需要为每一个 $i$ 求出在选取最优的 $k$ 时，最多能有多少个 $j$ 满足上述条件。

**输入格式：**
* 第一行包含测试用例的数量 $t$ ($1 \le t \le 100$)。
* 每个测试用例的第一行包含一个整数 $n$ ($1 \le n \le 5000$)。
* 第二行包含 $n$ 个整数 $a_1, a_2, \dots, a_n$ ($-10^9 \le a_i \le 10^9$)。
* 保证所有测试用例的 $n$ 之和不超过 $5000$。

**输出格式：**
* 对于每个测试用例，输出 $n$ 个整数，表示每个下标 $i$ 对应的答案。

---

### 2. 解题思路

**算法剖析：**

这道题的核心在于理解绝对值不等式 $|a_i - k| > |a_j - k|$ 的几何意义。在数轴上，这表示**点 $a_i$ 到点 $k$ 的距离，严格大于点 $a_j$ 到点 $k$ 的距离**。

我们需要选择一个 $k$，使得满足这个条件的 $j$ ($j > i$) 尽可能多。怎么选 $k$ 最划算呢？我们可以考虑两种极端情况：
1.  **让 $k$ 趋于正无穷大 ($k \to +\infty$)**：
    如果 $k$ 是一个极其巨大的正数，那么数轴上所有的 $a_i$ 和 $a_j$ 都在 $k$ 的左边。此时，谁的值越小，谁离 $k$ 就越远。
    因此，当选定一个极大的 $k$ 时，条件 $|a_i - k| > |a_j - k|$ 就等价于 **$a_i < a_j$**。也就是说，我们可以把所有位于 $i$ 右侧且值**严格大于** $a_i$ 的元素 $a_j$ 收集起来。
2.  **让 $k$ 趋于负无穷大 ($k \to -\infty$)**：
    同理，如果 $k$ 是一个极其巨大的负数，所有数都在 $k$ 的右边。此时，谁的值越大，谁离 $k$ 就越远。
    条件 $|a_i - k| > |a_j - k|$ 就等价于 **$a_i > a_j$**。也就是说，我们可以把所有位于 $i$ 右侧且值**严格小于** $a_i$ 的元素 $a_j$ 收集起来。

既然 $k$ 可以是任意整数，为了让满足条件的 $j$ 数量最大化，我们只需要在上述两种策略中选一个收益最大的即可！
* 策略 A 收益：统计 $i$ 右侧比 $a_i$ **大**的元素个数。
* 策略 B 收益：统计 $i$ 右侧比 $a_i$ **小**的元素个数。
* 最终答案就是 `max(策略A收益, 策略B收益)`。

**复杂度分析：**
* **时间复杂度**：外层循环遍历 $i$ (执行 $n$ 次)，内层循环遍历 $j$ (平均执行 $\frac{n}{2}$ 次)，因此单次测试用例的时间复杂度为 $O(n^2)$。由于题目保证所有测试用例的 $\sum n \le 5000$，最坏情况下的总计算量约为 $2.5 \times 10^7$ 次操作，在 1.5 秒的时间限制内绰绰有余。
* **空间复杂度**： $O(n)$，用于存储输入数组和答案数组。

### 3. 代码详解

```cpp
#include <bits/stdc++.h>
using namespace std;

void solve()
{
    int n;
    cin >> n;
    
    // a 数组存储输入的整数
    vector<long long> a(n);
    for (int i = 0; i < n; i++)
    {
        cin >> a[i];
    }
    
    // ans 数组用于存储每个位置的答案
    vector<int> ans(n, 0);
    
    // 外层循环：遍历每一个下标 i
    for (int i = 0; i < n; i++)
    {
        // smaller 记录 i 右侧比 a[i] 小的元素个数 (对应 k 取极小值的情况)
        // larger 记录 i 右侧比 a[i] 大的元素个数 (对应 k 取极大值的情况)
        int smaller = 0, larger = 0;
        
        // 内层循环：只看 i 右侧的元素 (j > i)
        for (int j = i + 1; j < n; j++)
        {
            if (a[j] < a[i])
                smaller++;   // 找到一个比 a[i] 小的数
            else if (a[j] > a[i])
                larger++;    // 找到一个比 a[i] 大的数
            // 注意：如果 a[j] == a[i]，无论 k 取什么值，距离都相等，不满足严格大于的条件，所以忽略。
        }
        
        // 当前位置的答案就是两种极端情况下的最大值
        ans[i] = max(smaller, larger);
    }
    
    // 格式化输出，最后一个元素后输出换行符，其余输出空格
    for (int i = 0; i < n; i++)
    {
        cout << ans[i] << (i + 1 < n ? " " : "\n");
    }
}

int main()
{
    // 优化 C++ 的输入输出速度，避免在大量 IO 时超时
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int t;
    cin >> t;
    while (t--)
    {
        solve();
    }
    return 0;
}
```

这道题的 $O(n^2)$ 解法已经非常完美地通过了测试。不过作为扩展思考，如果这道题的数据范围扩大到 $n \le 10^5$，我们就需要用到**树状数组 (Fenwick Tree)** 或 **线段树** 将复杂度优化到 $O(n \log n)$。
